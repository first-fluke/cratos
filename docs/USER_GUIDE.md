## 1. 시작하기

### 봇 찾기

1. Telegram 앱 열기
2. 검색창에 봇 유저네임 입력 (예: `@my_cratos_bot`)
3. 봇 선택 후 "시작" 버튼 클릭

### 첫 메시지

```
/start
```

봇이 인사 메시지로 응답하면 정상적으로 연결된 것입니다.

## 2. 기본 대화

Cratos는 자연어로 대화할 수 있습니다.

### 예시

**간단한 질문**
```
사용자: 안녕하세요
봇: 안녕하세요! 무엇을 도와드릴까요?
```

**정보 요청**
```
사용자: Python에서 리스트를 정렬하는 방법 알려줘
봇: Python에서 리스트를 정렬하는 방법은...
```

**코드 작성 요청**
```
사용자: 피보나치 수열을 계산하는 Python 함수 작성해줘
봇: def fibonacci(n):
    ...
```

## 3. 파일 작업

Cratos는 파일을 읽고, 쓰고, 목록을 조회할 수 있습니다.

### 파일 읽기

```
사용자: /home/user/readme.txt 파일 내용 보여줘
```

```
사용자: config.json 파일 읽어서 분석해줘
```

### 파일 쓰기

```
사용자: hello.txt 파일에 "Hello, World!" 저장해줘
```

```
사용자: 방금 만든 코드를 script.py로 저장해줘
```

### 디렉토리 목록

```
사용자: 현재 디렉토리 파일 목록 보여줘
```

```
사용자: /var/log 폴더에 뭐가 있어?
```


## 5. Git/GitHub 작업

Cratos는 Git 저장소 작업을 도와줍니다.

### Git 상태 확인

```
사용자: 현재 git 상태 알려줘
```

```
사용자: 최근 커밋 5개 보여줘
```

### 변경사항 확인

```
사용자: 변경된 파일 diff 보여줘
```

### 브랜치 관리

```
사용자: 현재 브랜치 뭐야?
```

```
사용자: feature/login 브랜치 생성해줘
```

### GitHub 연동

```
사용자: 이 저장소의 GitHub 이슈 목록 가져와줘
```

```
사용자: PR #42의 내용 요약해줘
```

## 6. 명령 실행

Cratos는 제한된 명령을 실행할 수 있습니다.

### 허용된 명령 예시

```
사용자: ls -la 실행해줘
```

```
사용자: pwd 실행해줘
```

```
사용자: echo "Hello" 실행해줘
```

### 주의사항

- 위험한 명령(rm -rf, 시스템 명령 등)은 승인이 필요합니다
- 일부 명령은 보안상 차단될 수 있습니다

## 7. 실행 기록 (리플레이)

Cratos는 모든 실행 기록을 저장하여 나중에 확인하고 재실행할 수 있습니다.

### 최근 실행 기록 조회

```
사용자: 최근 실행 기록 보여줘
```

### 특정 실행 상세 조회

```
사용자: 마지막 실행의 타임라인 보여줘
```

### 실행 재실행

```
사용자: 방금 실행을 다시 해줘
```

## 8. 팁과 주의사항

### 효과적인 사용 팁

1. **명확하게 요청하기**
   - ❌ "파일 좀 봐줘"
   - ✅ "/home/user/data.json 파일 읽어서 name 필드만 추출해줘"

2. **단계별로 요청하기**
   - 복잡한 작업은 여러 단계로 나눠서 요청하면 더 정확합니다

3. **경로 명시하기**
   - 파일 작업 시 전체 경로를 명시하면 오류가 줄어듭니다

### 주의사항

1. **민감한 정보**
   - API 키, 비밀번호 등 민감한 정보를 메시지로 보내지 마세요
   - 봇의 응답에 민감한 정보가 포함될 수 있으니 주의하세요

2. **비용**
   - LLM API 사용량에 따라 비용이 발생합니다
   - 불필요한 대화를 피하면 비용을 절감할 수 있습니다

3. **위험한 작업**
   - 파일 삭제, 시스템 명령 등 위험한 작업은 승인이 필요합니다
   - 승인 요청이 오면 내용을 꼼꼼히 확인 후 승인하세요

4. **응답 시간**
   - 복잡한 작업은 시간이 걸릴 수 있습니다
   - 보통 10초 내에 첫 응답이 옵니다
